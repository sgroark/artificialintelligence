Question 1

A*

Greedy Best-First

The A* algorithm utilizes the evaluation function (f) that takes both the actual cost (g) and heuristics (h) into consideration. In this case, one move costs 1 unit that is added to the g value of the cell, while the heuristic distance is calculated using the Manhattan distance. The algorithm then takes into account the summation of these values, which is then assigned to the value f (the evaluation function), and chooses the neighboring cells with the most reduced f value.

The Greedy Best-First algorithm, in contrast, only makes moves based on heuristic values of neighboring cells. The evaluation function for this search approach omits the actual cost (g) and only includes heuristic values (h). In this sense, the agent makes moves to neighboring cells with the lowest heuristic value. Resultantly, the agent moves closer towards the goal state without accruing the actual costs of the move, which explains why the agent above moves closer towards the goal state (heuristically) but has to redirect once encountering an obstacle.

In regular A*, the evaluation function is set to equal the sum of the actual cost and heuristic of the cell.

In Greedy Best-First, the evaluation function is set to equal just the value of the cell’s calculated heuristic.

Manhattan Distance functions efficiently for north, south, east, and west agent moves. It calculates the sum of the absolute difference of two points – the current cell being explored and the goal state – to define the heuristic.

Use of the Euclidean Distance allows for diagonal moves in addition to the standard N, S, E, W agent moves. It is used to calculate the shortest distance between two points (the current cell and the goal state).
